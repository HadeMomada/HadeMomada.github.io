Welcome to My Naming Standard

Hi I am Hade Momada and I want to explain here the process of developing my naming standard that I use for variables and function. (As a training to build a page in GitHub)

First Main Standard

Snake Standard (Year 1, Month 1)

In my opinion, the snake standard was the most similar to the normal writing mode, that's why I used this standard.
items_number | matrix | alpha_function()

Second Main Standard

CamelCase Standard (Year 1, Month 2)

Snake standard was a bit on my nerves because of the underlines, so I changed my standard to CamelCase.
itemsNumber | matrix | alphaFunction()

Showing Differences

Great Or Small In First Letter (Year 1, Month 4)

I want to be able to tell from the structure of the name whether the name is for a simple variable like int and string or a multiple variable like a list or dictionary or function at all, so I capitalized the first letter of multiple variables and functions And I put the first letter of simple variables in small form.
itemsNumber | Matrix | AlphaFunction()

Fix The Mismatch

Plural Words (Year 6, Month 5)

In some languages, writing the first letter of the name of the variable in lowercase or uppercase indicates whether the variable is public or private, which caused problems, so insteadof writing the first letter in uppercase for multiple variables, plural words and for the rest ofthem I used singular words.
itemsNumber | matrices | alphaFunction()

Making Sense

Type Name (Year 6, Month 6)

Writing the sum of multiple variables was sometimes not logical, like matrices, which are basically one matrix, not several, but since it was a list, it had to be written as a sum, so an alternative solution was necessary, which is why I used the name of the type of the variable itself.
iItemsNumber | l2Matrix | alphaFunction()
